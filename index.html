Best practices for effective AI-assisted development

Session Hygiene
/clear Between unrelated tasks. Clean reset beats lossy compact. Always.
/btw Quick side questions. Never enters context history. Zero token cost.
/rename Name sessions like branches: srh-uat-fixes, oauth-spec
claude --resume Pick up named sessions. --continue for the most recent one.
/compact <focus> Only when staying on the same task but context is getting heavy.
Esc + Esc Rewind to any checkpoint. Restore code, conversation, or both.
Workflow Habits
1 State work type upfront. "Quick question" / "feature spec" / "code session" / "QA run" — lets Claude calibrate effort and tools.
2 Interview pattern for big features. "Interview me about this feature using AskUserQuestion. Dig into edge cases and tradeoffs." Then write the spec in a fresh session.
3 Writer/Reviewer split. One session writes code. A fresh session reviews it. Fresh context = no self-bias.
4 Give verification criteria. Tests, screenshots, expected output. The single highest-leverage thing you can do. 2-3x quality
5 Explore > Plan > Implement > Commit. Use Plan Mode (Shift+Tab) for unknowns. Skip the ceremony for small fixes.
6 Two corrections max. If Claude fails twice on the same thing, /clear and rewrite the prompt with what you learned.
Token Efficiency
Max 2 parallel agents. Default. Bypass when you have high token budget before reset.
/effort medium for routine work. /effort high for deep sessions.
Subagents for research. They explore in separate context, report back summaries only. Main stays clean.
CLI > MCP. gh, curl, jq — no schema overhead. MCP only when CLI can't.
Scope investigations. "Check src/auth/" not "investigate the codebase". Narrow = fewer tokens.
From Thariq @ Anthropic
Skills are folders, not just files. Include scripts, data, assets alongside SKILL.md. Claude discovers them progressively.
Gotchas section = highest signal. Build up from failure points over time. Update after every recurring mistake.
File system = context engineering. Structure for progressive disclosure, not flat dumps. Let Claude pull what it needs.
Claude Code is a general agent. Not just for code — use it for research, planning, analysis, communication, anything.
CLAUDE.md Rules of Thumb
For each line ask: "Would removing this cause Claude to make mistakes?" If not, cut it.
If Claude ignores a rule, the file is too long and the rule is getting lost. Prune, don't add emphasis.
Hooks for musts, CLAUDE.md for shoulds. Advisory (80% adherence) vs deterministic (100%).
Don't document what Claude can infer. It reads your code. Only add what it can't figure out alone.
