What's New Latest

Channels (Mar 19) — --channels flag (research preview) lets MCP servers push messages into your session. Enables remote interaction via channel plugins. Admins control access with allowedChannelPlugins managed setting.

+

/remote-control (VS Code only, Mar 18) — Bridges your VS Code session to claude.ai/code so you can continue from a browser or phone. Separate from --channels.

+

Sonnet 4.6 — New balanced model. Better agentic search, fewer tokens. Use for shell fixes, quick tasks. Opus 4.6 remains default for deep work.

+

Plugin System — Extend Claude Code with community plugins: custom commands, agents, hooks, MCP servers. Browse via /plugin. Code intelligence plugins give Claude LSP-level diagnostics.

+

Compaction API (beta) — Server-side context summarization. Also: "Summarize from here" via Esc Esc lets you partially compact without starting over.

+

MCP Elicitation (Mar 14) — MCP servers can request structured input mid-task via interactive dialogs. New Elicitation and ElicitationResult hooks to intercept and override.

Recent — still worth knowing

•

Auto Mode (Mar 24) — Classifier model handles permission decisions with safety guardrails. Blocks destructive actions, lets routine work proceed. Team plan now. Enable: claude --enable-auto-mode or Shift+Tab.

•

Voice Mode — Push-to-talk (hold spacebar, rebindable via voice:pushToTalk in keybindings.json). Dictate specs, describe bugs, brainstorm hands-free.

•

Opus 4.6 + 1M Context (Mar 13) — 1M context GA on Max/Team/Enterprise. Output cap: 64K tokens default, 128K max. Opus defaults to medium effort. /fast toggles faster output, same model.

•

Sparse Worktrees (Mar 14) — worktree.sparsePaths setting for large monorepos. Check out only the directories you need via git sparse-checkout.

•

Transcript Search (Mar 25) — Press / in transcript mode (Ctrl+O) to search, n/N to step through matches.

•

Conditional Hooks (Mar 26) — if field for hooks uses permission rule syntax (e.g., Bash(git *)) to filter when they fire. Reduces process spawning overhead.

/effortlow medium high. Or auto — Claude picks per-turn.
Shift+TabCycle permission modes: default → acceptEdits → plan → auto.
/renameName sessions like branches. Resume later with --resume.
/loopRecurring tasks: /loop 5m check deploy. Session-scoped, dies on exit.
/rewindOr Esc Esc. Restore code, conversation, or both. "Summarize from here" trims context.
-nCLI flag: claude -n "my-session" sets display name at startup.

Workflow

1

State work type upfront. "Quick question" / "feature spec" / "code session" / "QA run". Sets expectations both ways.

2

Interview before building. Have Claude ask you the hard questions first. Then spec in a fresh session.

3

Writer/Reviewer split. One session writes. A fresh one reviews. No self-bias. Code Review agent now does this automatically on PRs.

4

Give verification. Tests, screenshots, expected output. Install LSP plugins — they give Claude automatic diagnostics after every edit. Highest-impact plugin you can install.

5

Explore > Plan > Implement > Commit. Plan Mode (Shift+Tab) for unknowns. Auto Mode for trusted execution. Skip for trivial fixes.

6

Two corrections max. Third fail = /clear and rewrite the prompt. Don't Goodhart — optimize for clarity, not length.

Daily Rituals — structure that compounds

AM

Morning alignment. Pull yesterday's wins, open reminders, Linear issues, GitHub PRs. Energy check (sharp/steady/recovering). Generate 1-3-5 plan with a protected thinking block. Push to Reminders.

PM

Evening review. Lead with wins, not scores. Honest reflection: what shipped, what's blocked, which products got attention. Process carry-overs (winding down vs night session). Prep tomorrow's draft.

FR

Friday pulse. 5-line week retro: product attention days, top 3 shipped items, bounced-all-week items, Monday leverage candidate. Prevents Monday blindness.

The leverage question: What's the one thing today that, if done, makes everything else easier or irrelevant? That gets the deep block. Everything else fits around it.

Scheduling — delegate the recurring to the machine

~

/loop for intervals. /loop 5m /babysit-prs, /loop 1h check deploy. Defaults to 10m. Session-scoped — dies when you exit.

~

CronCreate for precision. Standard 5-field cron in local time. "0 9 * * 1-5" = weekdays at 9am. Pass durable: true to survive restarts.

~

One-shot reminders. recurring: false — fires once, then auto-deletes.

~

7-day auto-expiry. Recurring tasks self-destruct after 7 days. Durable tasks persist to .claude/scheduled_tasks.json.

~

Dreaming: off by default. autoDreamEnabled: false in settings.json. Enable only if you have a specific idle-time use case. Tokens burn continuously otherwise.

Tokens — every wasted token is a thought that never happened

$

Max 2 parallel agents. Each loads ~7K overhead. Bypass only with high budget.

$

CLI > MCP. gh curl jq — no schema bloat.

$

ToolSearch saves tokens. 50+ tool servers load on demand, not upfront.

$

Subagents for research. Separate context. Summary back.

$

Scope narrowly. "Check src/auth/" not "investigate".

$

Skills over MCPs. Transparent, reviewable, no black boxes.

Knowledge Architecture

Two domains, one rule. ~/Documents/Projects/ is yours — structured docs, specs, reports. ~/Workspaces/ is the agent's — dev work, worktrees, tooling. Neither pollutes the other.

1

CLAUDE.md — Identity & instructions. Who you are, tech stack, communication style. The onboarding doc AI reads in 2 seconds.

2

heartbeat.md — What's in progress right now. Active deployments, current sprint, blockers. Update every session.

3

MEMORY.md — Index of persistent learnings. Each memory is a separate .md file. Cap at 15, merge aggressively.

4

decisions.md — The "why" behind choices. Why approach A over B. Future-you will thank you.

5

Project folder convention — Every project: docs/ (deliverables), context/ (inputs), data/ (exports), media/ (videos, images). Known structure = direct lookup, not search.

Context Bloat — AI burnout is real

!

Long threads = worse output. Token cost rises, hallucinations creep in. Compaction API helps but doesn't eliminate this.

!

Close, don't continue. When a conversation gets long, close it. Start fresh. Update CLAUDE.md with learnings.

!

"Summarize from here" — New: partial conversation summarization. Trim the fat without starting over.

!

Invest in docs, not prompts. 1.5 days of clear documentation > 2 weeks of prompt trial-and-error.

Avoid — coordination failures start small

X

Kitchen sink sessions. One task drifts into three. /clear between.

X

Correction loops. 3+ corrections = polluted context. Rewrite the prompt.

X

Bloated CLAUDE.md. Over 5KB and rules get lost. Move knowledge to skills.

X

Unscoped exploration. "Investigate everything" fills context. Scope or subagents.

X

Ship without verification. Plausible != correct. No tests, no confidence.

X

Dreaming without purpose. Auto-dream burns tokens continuously. Disable unless you have a specific polling use case.
