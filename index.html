The middle path between clicking "approve" 50 times and yolo-ing with skip-permissions. Enable via Shift+Tab or --permission-mode auto. Your safety hooks still run on top.

Workflow

1

State work type upfront. "Quick question" / "feature spec" / "code session" / "QA run".

2

Interview before building. Have Claude ask you the hard questions first. Then spec in a fresh session.

3

Writer/Reviewer split. One session writes. A fresh one reviews. No self-bias.

4

Give verification. Tests, screenshots, expected output. Highest-leverage habit you can build.

5

Explore > Plan > Implement > Commit. Plan Mode for unknowns. Skip for trivial fixes.

6

Two corrections max. Third fail = /clear and rewrite. Don't Goodhart your prompts — optimize for clarity, not length.

Tokens — every wasted token is a thought that never happened

$

Max 2 parallel agents. Each loads ~7K overhead.

$

CLI > MCP. gh curl jq — no schema bloat.

$

Disable unused plugins. 300-500 tokens each.

$

Subagents for research. Separate context. Summary back.

$

Scope narrowly. "Check src/auth/" not "investigate".

$

No redundant reads. Already read? Skip unless edited.

Setup — your config is your prior

*

CLAUDE.md: less is more. Would removing this cause mistakes? No? Cut it.

*

Hooks for musts. CLAUDE.md ~80% adherence. Hooks = 100%.

*

Memory cap: 15 files. Merge before adding. Index only.

*

Skills are folders. Scripts, data, assets. Progressive disclosure.

*

Gotchas = gold. Build from failures. Update after every mistake.

*

Push CLAUDE.md to git. Shared context compounds over time.

Avoid — coordination failures start small

X

Kitchen sink sessions. One task drifts into three. /clear between.

X

Correction loops. 3+ corrections = polluted context. Rewrite the prompt.

X

Bloated CLAUDE.md. Over 5KB and rules get lost. Move knowledge to skills.

X

Unscoped exploration. "Investigate everything" fills context. Scope or subagents.

X

Ship without verification. Plausible != correct. No tests, no confidence.
