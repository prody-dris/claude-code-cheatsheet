Workflow

1

State work type upfront. "Quick question" / "feature spec" / "code session" / "QA run". Sets expectations both ways.

2

Interview before building. Have Claude ask you the hard questions first. Then spec in a fresh session.

3

Writer/Reviewer split. One session writes. A fresh one reviews. No self-bias. Code Review agent now does this automatically on PRs.

4

Give verification. Tests, screenshots, expected output. Install LSP plugins — they give Claude automatic diagnostics after every edit. Highest-impact plugin you can install.

5

Explore > Plan > Implement > Commit. Plan Mode (Shift+Tab) for unknowns. Auto Mode for trusted execution. Skip for trivial fixes.

6

Two corrections max. Third fail = /clear and rewrite the prompt. Don't Goodhart — optimize for clarity, not length.

Scheduling — delegate the recurring to the machine

~

/loop for intervals. /loop 5m /babysit-prs, /loop 1h check deploy. Defaults to 10m if no interval given. Session-scoped — dies when you exit.

~

CronCreate for precision. Standard 5-field cron in local time. "0 9 * * 1-5" = weekdays at 9am. Pass durable: true to survive restarts.

~

One-shot reminders. recurring: false — fires once, then auto-deletes. Pin minute/hour/day to specific values.

~

7-day auto-expiry. Recurring tasks self-destruct after 7 days. Bounds runaway loops. Durable tasks persist to .claude/scheduled_tasks.json.

~

Desktop /schedule. GUI-based in Claude Desktop / Cowork. Persists without machine being awake (cloud-backed on paid plans).

Knowledge Architecture

The .context folder pattern. Every project repo gets a .context/ directory. This is how AI remembers across sessions. Structure it once, benefit forever.

1

CLAUDE.md — Identity & instructions. Who you are, tech stack, communication style, who you serve. The onboarding doc AI reads in 2 seconds.

2

heartbeat.md — What's in progress right now. Active deployments, current sprint, blockers. Update every session. AI picks up where you left off.

3

MEMORY.md — Index of persistent learnings. Each memory is a separate .md file. Append-only — mark superseded, never delete.

4

decisions.md — The "why" behind choices. Why approach A over B. Future-you will thank you.

5

SECURITY.md — Non-negotiable guardrails. No secrets in code, parameterised queries, permissions on every DocType. Goes in every repo.

6

glossary.md — Domain-specific vocabulary. Prevents AI from guessing wrong.

Context Bloat — AI burnout is real

!

Long threads = worse output. Token cost rises, hallucinations creep in. Same task, different quality at conversation start vs end.

!

Close, don't continue. When a conversation gets long, close it. Start fresh. Update CLAUDE.md with learnings. 2 minutes of updates saves 20 minutes of re-explaining.

!

UI reads, CLI writes. Claude UI can read your GitHub repos. But for writing (commits, PRs, file updates), you need Claude Code / CLI. UI is texting; CLI is pair programming.

!

Invest in docs, not prompts. 1.5 days of clear documentation > 2 weeks of prompt trial-and-error. Structure your input so AI works on the first attempt.

Avoid — coordination failures start small

X

Kitchen sink sessions. One task drifts into three. /clear between.

X

Correction loops. 3+ corrections = polluted context. Rewrite the prompt.

X

Bloated CLAUDE.md. Over 5KB and rules get lost. Move knowledge to skills.

X

Unscoped exploration. "Investigate everything" fills context. Scope or subagents.

X

Ship without verification. Plausible != correct. No tests, no confidence.
