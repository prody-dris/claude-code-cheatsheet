Claude Code Cheatsheet

Best practices, setup patterns, and workflow habits for Claude Code — from real daily use as a non-technical PM running multiple products.
Last updated: 25 March 2026
Foundations
Session Hygiene
/clear Between unrelated tasks. Clean reset beats lossy compact. Always.
/btw Quick side questions. Never enters context history. Zero token cost.
/rename Name sessions like branches: srh-uat-fixes, oauth-spec
claude --resume Pick up named sessions. --continue for the most recent one.
/compact <focus> Only when staying on the same task but context is getting heavy.
Esc + Esc Rewind to any checkpoint. Restore code, conversation, or both.
/effort medium For routine work. /effort high for deep sessions. Toggle to save thinking tokens.
Workflow Habits
1 State work type upfront. "Quick question" / "feature spec" / "code session" / "QA run" — lets Claude calibrate effort and tools.
2 Interview pattern for big features. "Interview me about this feature using AskUserQuestion. Dig into edge cases and tradeoffs." Then write the spec in a fresh session.
3 Writer/Reviewer split. One session writes code. A fresh session reviews it. Fresh context = no self-bias.
4 Give verification criteria. Tests, screenshots, expected output. The single highest-leverage thing you can do. 2-3x quality
5 Explore > Plan > Implement > Commit. Use Plan Mode (Shift+Tab) for unknowns. Skip the ceremony for small fixes.
6 Two corrections max. If Claude fails twice on the same thing, /clear and rewrite the prompt with what you learned.
Token Efficiency
Spend Less
Max 2 parallel agents. Each loads ~7K tokens of system prompt overhead. Bypass only when token budget is high before reset.
CLI > MCP. gh, curl, jq have no schema overhead. MCP tools load definitions into every context.
Disable unused plugins. Playwright, Linear, skill-creator — disable when not needed. Each adds ~300-500 tokens of tool schemas.
Targeted Playwright snapshots. DOM snapshots are 2-4K tokens each. Use targeted selectors, not full-page, unless debugging layout.
Spend Smart
Subagents for research. They explore in separate context, report back summaries only. Main conversation stays clean.
Scope investigations. "Check src/auth/" not "investigate the codebase". Narrow = fewer tokens consumed.
No redundant file reads. If Claude read a file this session, it doesn't need to read it again unless it was edited.
Never re-dispatch failed agents. If agents hit rate limits or errors, start a fresh session instead of retrying.
Setup That Pays Off
CLAUDE.md — Your Persistent Brain
1 For each line ask: "Would removing this cause Claude to make mistakes?" If not, cut it. Bloated files cause Claude to ignore your actual instructions.
2 Hooks for musts, CLAUDE.md for shoulds. Hooks are deterministic (100% enforcement). CLAUDE.md is advisory (~80%). Put safety rules in hooks.
3 Don't document what Claude can infer. It reads your code. Only add what it can't figure out alone — build commands, conventions, gotchas.
4 If Claude keeps ignoring a rule, the file is too long and the rule is getting lost. Prune first, add emphasis second.
5 Check it into git. CLAUDE.md compounds over time. Your team benefits from shared context. Push updates after every session.
Hooks — Your Safety Net
PreToolUse: credential-guard.py — Blocks writing tokens/keys to non-allowed files. Matches Bash, Write, Edit.
PreToolUse: pre-tool-use.sh — Blocks sudo, rm -rf, force push, push to main, git reset --hard, DROP, bench destroy.
PreToolUse: agent-safety-gate.py — Auto-injects safety rules into every subagent dispatch.
PostToolUse: audit logger — Logs every file write and bash command to daily log. 30-day rotation.
Memory — Stay Lean
Cap at 15 files per scope. Merge or delete before adding new ones. Memory bloat = wasted tokens every session.
MEMORY.md is an index, not content. One line per file. Under 20 lines. It loads every conversation.
Never store credentials in memory. Use project CLAUDE.md or .env files (chmod 600). Memory files reference WHERE, never WHAT.
Delete stale memories. If it references something that no longer exists, kill it. No nostalgia.
Folder Structure — Separation of Concerns
~/Workspaces/ Code repos, tools, configs — Claude's working projects. Each has its own CLAUDE.md.
~/Documents/Projects/ Your working documents — specs, legal, references. Human-authored, not code.
~/.claude/agents/ Custom subagents. Use skill-selector when unsure which to route to.
~/.claude/hooks/ Safety hooks — credential guard, destructive command blocker, agent safety gate, audit logger.
~/.claude/projects/*/memory/ Scoped memory files. Each project scope gets its own set. Cap at 15.
Daily Rituals
CC Morning & Evening
AM CC Morning — daily plan generator. Pulls NeoSapien conversations, calendar, reminders, and pending items. Generates a prioritized daily plan with time blocks. Start every workday here.
PM CC Evening — daily review & tomorrow prep. Reviews what got done, updates reminders, flags carry-overs, and preps tomorrow's context. End every workday here.
* One task per session. Morning plan gives you the list. Execute each item in its own session. /clear between. No context bleed.

Official Best Practices
Verification is #1. Give Claude tests, expected output, screenshots. It catches its own mistakes — you become the director, not the debugger.
Let Claude interview you. For big features, start with "interview me about this." Claude surfaces edge cases you haven't considered.
Pipe data in. cat error.log | claude is faster than pasting. Use @file to reference files in prompts.
Run non-interactive in CI. claude -p "prompt" for automated pipelines — test fixing, code review, doc generation.
Anti-Patterns — Stop Doing These
X Kitchen sink sessions. Start with one task, drift to another, then back. Context fills with irrelevant noise. Use /clear.
X Correction loops. Correcting Claude 3+ times in a row. Context is polluted with failed approaches. /clear and rewrite the prompt.
X Bloated CLAUDE.md. If it's over 5KB, Claude starts ignoring rules. Prune ruthlessly. Move domain knowledge to skills (loaded on demand).
X Infinite exploration. "Investigate the codebase" without scope. Claude reads hundreds of files, fills context. Always scope narrowly or use subagents.
X Trust without verification. Claude produces plausible code that misses edge cases. No tests = no confidence. Always verify before shipping.
